# UI/UX 가이드라인: Workspace 통합 검색 및 Admin 대시보드 개편

## 1. 개요
본 문서는 LogiVOC 시스템의 `prd-workspace-admin-dash.md`를 기반으로, Workspace 탭의 통합 검색 포털 및 System Admin 탭의 대시보드 개편을 위한 프론트엔드 UI/UX 설계 가이드라인입니다.

## 2. WorkspacePage (통합 검색 포털) UI/UX 설계
기존 더미 데이터를 걷어내고 실제 데이터 기반의 AI 시맨틱 검색, 다중 필터링, 구조화된 문서 뷰어를 제공하는 레이아웃입니다.

### 2.1 전체 레이아웃 구성
- **상단 (Search & Filter Section)**
  - **통합 검색바**: 화면 중앙 상단에 크고 명확하게 배치. 사용자가 바로 입력할 수 있도록 유도하는 Placeholder 텍스트 적용 (예: "검색어 또는 질문을 입력하세요...").
  - **다중 필터 패널**: 검색바 하단 또는 우측에 드롭다운/버튼 그룹 형태로 배치.
    - **카테고리 필터**: 다중 선택이 가능한 드롭다운(Select).
    - **상태 필터**: 칩(Chip) 또는 토글 버튼 형태 (승인, 반려, 대기 등).
    - **날짜 필터**: 시작일과 종료일을 지정할 수 있는 Date Picker.
- **하단 (Search Results Section)**
  - **결과 리스트**: 총 검색 결과 건수 표기.
  - **검색 결과 카드**:
    - 문서 제목을 굵게 표시.
    - **AI 요약본**: 카드 본문에 1~2줄의 텍스트로 미리보기 제공.
    - 카테고리 뱃지 및 상태 뱃지 표시.
    - 작성 날짜 표시.
    - AI 시맨틱 검색에 의한 결과일 경우, 우측 상단에 **연관도(유사도 점수)** 표시 (예: 98% 일치).

### 2.2 문서 상세 뷰어 (Document Detail Viewer)
- 검색 결과 클릭 시, 페이지 이동 없이 화면 우측에서 나타나는 **슬라이드 오버(Slide-over) 패널** 또는 **대형 모달(Modal)** 채택. 사용자 탐색 맥락 유지를 위해 화면 전환 최소화.
- **상세 뷰어 내부 구성**:
  - **상단 헤더**: 문서 제목, 카테고리/상태 뱃지, 닫기 버튼.
  - **AI 요약본 (Summary)**: 뷰어 진입 시 가장 먼저 눈에 띄도록 최상단의 하이라이트 박스 영역에 배치.
  - **구조화된 데이터 뷰어 (Structured Data Viewer)**:
    - 추출된 JSON 데이터를 가독성 높은 **Markdown Table** 형식(Key-Value 형태)으로 렌더링.
    - 테이블 우측 상단에 **'Table 보기 / JSON 보기' 토글 스위치**를 제공하여, 개발자나 상세 확인이 필요한 사용자가 원본 JSON 형태로도 볼 수 있도록 지원.
  - **원본 텍스트**: 가장 하단에 배치하며, 내용이 길 경우 아코디언(Accordion) 형식으로 기본 접어두기 적용.

## 3. SystemAdminPage (대시보드) UI/UX 설계
기존 'Coming Soon'이었던 Dashboard 탭을 활성화하여 시스템 및 지식 베이스 현황을 직관적으로 보여주는 화면입니다.

### 3.1 대시보드 레이아웃
- 가독성이 좋은 **그리드 레이아웃(Grid Layout)** 기반의 위젯형 카드 UI 적용. 화면 크기에 따라 열 개수가 유연하게 조절되는 반응형 구성.

### 3.2 핵심 위젯 구성
1. **요약 지표 (KPI) 위젯 (최상단 행)**
   - 3~4개의 텍스트 기반 카드형 위젯 가로 배치.
   - **총 문서 수**: 아이콘과 함께 큰 숫자로 누적치 표기.
   - **승인 완료 문서 수**: 사용/조회 가능한 승인 문서 현황.
   - **대기/반려 문서 수**: 리뷰가 필요한 상태의 현황 표기 (필요시 알림 컬러 적용).
2. **카테고리별 통계 차트 위젯 (중앙 좌측)**
   - **UI 형태**: 파이 차트(Pie Chart) 또는 도넛 차트(Doughnut Chart).
   - **동작**: 범례(Legend)를 측면 또는 하단에 배치. 마우스 호버(Hover) 시 툴팁을 띄워 상세 개수 및 비율(%) 표시.
3. **최근 활동 내역 (Recent Activities) 위젯 (중앙 우측)**
   - **UI 형태**: 세로형 타임라인(Timeline) 리스트.
   - **항목 구성**:
     - 이벤트 종류별(등록, 승인, 상태 변경) 고유 아이콘 및 색상 적용.
     - 주요 내용 요약 노출 (예: "[마케팅] 신규 프로모션 기획안 등록").
     - 상대적 시간 표기 (예: "방금 전", "2시간 전", "1일 전").

## 4. 공통 UI 컴포넌트 및 가이드
- **스타일링**: LogiVOC 시스템의 디자인 시스템 테마 색상(Primary Color, Gray Scale) 상속. 일관된 `border-radius`, `shadow` 적용.
- **상태 처리 (Loading & Empty States)**:
  - **로딩 중 (Loading)**: 화면 전체를 덮는 스피너보다는 부드러운 스켈레톤(Skeleton) UI를 사용하여 사용자가 대기 시간을 짧게 느끼도록 함.
  - **결과 없음 (Empty State)**: 검색 결과나 최근 활동이 없을 경우 중앙에 아이콘과 함께 명확한 안내 텍스트 제공 (예: "조건에 맞는 검색 결과가 없습니다.").
- **반응형 (Responsive)**: 모바일 및 태블릿 뷰어포트 대응. 모바일에서는 하단 뷰어가 풀 사이즈 시트로, 그리드는 1열(세로) 배치로 전환되도록 구현.
