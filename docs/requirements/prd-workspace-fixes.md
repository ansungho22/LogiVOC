# Product Requirements Document (PRD) - 워크스페이스 및 시스템 개선

## 1. 개요
본 PRD는 사용자 피드백을 반영하여 시스템의 화면, 네이밍 통일성, 관리자 페이지 구체화, 그리고 전체적인 폴더 구조 분리를 통해 시스템의 품질과 유지보수성을 향상시키기 위한 요구사항을 정의합니다.

## 2. 목표
- 탭 및 라우팅 오류 수정을 통한 명확한 사용자 경험 제공
- 미구현된 관리자 페이지(Dashboard, User Management) 기능 구현
- 탭 네이밍의 다국어(영어) 통일성 확보
- 프론트엔드 및 백엔드 폴더 구조 리팩토링을 통한 코드 가독성 및 유지보수성 향상

## 3. 상세 요구사항

### 3.1 화면 및 탭 이슈 개선 (워크스페이스 & 데이터 등록)
- **현상:** 데이터 등록 탭과 워크스페이스 탭의 화면이 동일하게 렌더링되는 문제 발생.
- **요구사항:**
  - 라우터 설정(React Router 등)을 점검하여 올바른 컴포넌트가 마운트되도록 수정.
  - 두 화면의 레이아웃, 컬러, 아이콘 등의 시각적 차별화를 강화하여 사용자가 현재 위치를 명확히 인지할 수 있도록 개선.

### 3.2 탭 네이밍 통일
- **현상:** 메인 탭 3개 중 '데이터 등록'이 한글로 되어 있어 언어적 통일성이 떨어짐.
- **요구사항:**
  - 모든 메인 탭 네이밍을 영어로 통일.
  - 예시: 
    - `Data Registration` (기존 '데이터 등록')
    - `Workspaces` (기존 'Workspace')
    - `Admin` (기존 'Admin')
  - 탭의 역할이 명확히 드러나도록 툴팁 또는 서브 타이틀 제공.

### 3.3 관리자(Admin) 페이지 기능 구현
- **현상:** Dashboard와 User Management 탭이 'Coming Soon' 상태로 미구현 상태임.
- **요구사항:**
  - **Dashboard:** 시스템 요약 정보를 보여주는 위젯(예: 총 워크스페이스 수, 최근 등록된 데이터 수 등)을 목업(Mock) 데이터를 활용하여 구현. 향후 실제 API와 연동 가능하도록 컴포넌트 구조 설계.
  - **User Management:** 사용자 목록, 권한 설정(Role)을 보여주는 테이블 형태의 UI 구현. 목업 데이터를 통해 페이징, 검색 기능 등을 검증할 수 있도록 구성.

### 3.4 폴더 및 구조 리팩토링
- **현상:** 백엔드와 프론트엔드 코드 파일들이 목적별로 명확히 분리되지 않고 혼재됨.
- **요구사항:**
  - **백엔드 (Backend):**
    - `app/models/`: 데이터베이스 모델 정의 파일
    - `app/schemas/`: Pydantic 스키마 정의 (요청/응답 모델)
    - `app/crud/`: 데이터베이스 CRUD 연산 로직
    - `app/api/` 또는 `app/routers/`: API 엔드포인트(라우터) 정의
  - **프론트엔드 (Frontend):**
    - `src/api/`: 백엔드 통신 API 호출 함수 모음
    - `src/types/`: TypeScript 타입 및 인터페이스 정의
    - `src/components/`, `src/pages/`: UI 컴포넌트 및 페이지 모음
  - 리팩토링 후 각 모듈 간의 의존성 오류(Import 에러)가 발생하지 않도록 면밀히 검증.

## 4. 제약 사항 및 고려 대상
- 구조 변경 과정에서 기존 기능(워크스페이스 목록 조회, 데이터 업로드 등)이 정상 작동하는지 회귀 테스트(Regression Test) 필수.
- 향후 Phase 2(아키텍처 및 디자인 설계)에서 관련 화면에 대한 상세 UI/UX 명세와 디렉터리 트리(아키텍처)를 업데이트해야 함.
