# PRD: 시스템 UI 및 기능 개편 (Workspace 통합 검색 및 Admin 대시보드)

## 1. 개요
본 문서는 LogiVOC 시스템의 UI 및 기능 개편을 위한 제품 요구사항 정의서(PRD)입니다. 사용자의 요청에 따라 Workspace 탭을 AI 기반의 통합 검색 포털로 개편하고, System Admin 탭의 'Coming Soon' 상태였던 대시보드를 활성화하여 지식 베이스 현황을 직관적으로 파악할 수 있도록 관리 환경을 고도화하는 것을 목표로 합니다.

## 2. 주요 목표
- **Workspace 탭 (통합 검색 포털) 개편**: 더미 데이터를 제거하고, `pgvector`를 활용한 AI 시맨틱(의미론적) 검색 기능을 도입. 강력한 필터링과 편리한 문서 상세 뷰어를 제공하여 탐색 경험을 개선.
- **System Admin 탭 (통계 대시보드) 개편**: 지식 베이스의 전체적인 등록, 승인 현황 및 최신 활동을 시각화하는 대시보드 구현.

## 3. 기능 요구사항

### 3.1 Workspace 탭 (통합 검색 포털) 개편
1. **더미 데이터 제거 및 실 데이터 연동**
   - 하드코딩된 더미 데이터를 모두 제거하고, 데이터베이스에서 실제 데이터를 불러오도록 아키텍처 변경.
2. **AI 시맨틱 검색 기능 도입**
   - 단순 텍스트 매칭이 아닌 문맥과 의미를 이해하는 검색 기능 제공.
   - `pgvector` 기반의 벡터 데이터베이스를 활용하여 사용자 쿼리와 문서의 임베딩 유사도 계산 및 결과 반환.
3. **다중 필터링 시스템 구축**
   - 검색 조건 세분화 기능 제공.
   - **필터링 조건**: 문서 카테고리, 문서 등록 상태(승인, 반려, 대기 등), 작성 날짜.
   - 검색어(시맨틱 검색)와 필터링 조건의 복합 조회(AND 조건) 지원.
4. **구조화된 문서 상세 뷰어 (Viewer) 제공**
   - 검색 결과 리스트에서 특정 문서 클릭 시, 모달 또는 상세 페이지 뷰어를 표시.
   - **뷰어 구성 요소**:
     - **원본 텍스트**: 문서 원문 조회.
     - **AI 요약본**: AI를 통해 생성된 문서의 핵심 요약 내용.
     - **구조화된 데이터 시각화**: 원문에서 추출된 데이터(JSON 포맷)를 가독성이 좋은 Table 형태 또는 Key-Value 쌍 UI로 시각화하여 렌더링.

### 3.2 System Admin 탭 (통계 대시보드) 개편
1. **대시보드 서브 탭 활성화**
   - 기존의 'Coming Soon' 안내를 제거하고, Dashboard UI를 활성화.
2. **지식 베이스 통계 위젯 제공**
   - **핵심 지표 (KPI)**: 전체 등록 문서 수, 최종 승인 완료된 문서 수 표기.
   - **카테고리별 통계**: 카테고리별 문서 분포를 직관적으로 파악할 수 있도록 차트(파이/도넛 차트 등) 제공.
   - **최근 활동 내역 (Recent Activities)**: 새 문서 등록, 승인, 상태 변경 등 시스템 내 최근 주요 이벤트를 타임라인 형태의 리스트로 노출.

## 4. 프론트엔드 UI/UX 변경 사항
- **Workspace (검색 및 리스트 페이지)**
  - 상단 영역에 대형 검색바 및 필터 토글 UI 배치.
  - 좌측 또는 상단에 카테고리, 상태 기반의 체크박스/드롭다운 필터 패널 추가.
  - 검색 결과 카드는 문서의 요약 정보, 카테고리 뱃지, 연관도(시맨틱 점수) 등을 표기.
- **Workspace (문서 상세 뷰어)**
  - 가독성을 고려한 모달 혹은 슬라이드 오버(Slide-over) 패널 도입.
  - 요약본 영역 강조, 구조화된 JSON 데이터의 Table 변환 컴포넌트 추가 개발.
- **System Admin (Dashboard)**
  - 그리드 레이아웃(Grid Layout) 기반의 위젯형 UI 구현.
  - Recharts 등 차트 라이브러리를 활용한 데이터 시각화.

## 5. 백엔드 API 연동 필요 사항 (신규 추가 및 수정)
- **Workspace (통합 검색) API**
  - `GET /api/documents/search`
    - 파라미터: `query`(검색어), `category`, `status`, `page`, `limit` 등.
    - 기능: pgvector를 이용한 벡터 유사도 검색 및 필터 조건에 맞는 데이터 리스트 반환.
  - `GET /api/documents/{id}`
    - 기능: 특정 문서의 상세 데이터(원본, 요약 텍스트, 구조화된 JSON 데이터) 반환.
- **Admin Dashboard API**
  - `GET /api/admin/stats/overview`
    - 기능: 총 문서 수, 상태별(대기, 승인 등) 문서 수 등 요약 지표 리턴.
  - `GET /api/admin/stats/categories`
    - 기능: 카테고리별 문서 갯수 통계 리턴 (차트 렌더링용).
  - `GET /api/admin/activities/recent`
    - 기능: 문서 및 지식 베이스 관련 최근 작업 로그/활동 이력 리턴.

## 6. 향후 계획 및 검증
- 해당 기획 문서를 기반으로 `architect` 에이전트와 `ui_ux_designer` 에이전트가 기술 설계 및 UI/UX 설계를 이어서 진행합니다.
- 변경된 아키텍처와 API 스펙, 화면 설계서가 완료되면, 개발 에이전트들이 구현을 시작합니다.
